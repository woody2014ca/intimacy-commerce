# 功能对照与网站功能描述（完整版）

依据 README 与项目设计，以下为**已实现**与**可选扩展**的对照。

---

## 一、项目定位（README）

- **亲密关系教育电商平台**：电商系统，非多级营销，所有佣金仅来源于商品销售。
- **形态**：Web H5 前台 + Admin 后台 + API 服务。
- **推荐**：仅两级（Level1/Level2），代码层锁死 MAX_REF_DEPTH=2。
- **合规**：佣金仅订单 **COMPLETED** 时结算；冻结期可配置；提现人工审核。

---

## 二、后端 API 能力（已具备）

| 能力 | 接口 | 说明 |
|------|------|------|
| 登录/注册/刷新 | POST /auth/login, register, refresh | 手机/邮箱、开发环境超级管理员兜底 |
| 仪表盘 | GET /admin/dashboard | 订单数、用户数、商品数、GMV |
| 用户列表 | GET /admin/users | 角色、审核、邀请码等 |
| 分类 | GET /admin/categories | 商品分类下拉 |
| 商品 CRUD | GET/POST/PATCH /admin/products | 列表、新增、编辑（含上下架） |
| 订单列表/审核 | GET /admin/orders, PATCH /admin/orders/:id | 订单列表、标记完成等 |
| 佣金列表 | GET /admin/commissions | 全部佣金记录 |
| 提现列表/审核 | GET /admin/withdrawals, PATCH /admin/withdrawals/:id | 提现申请、通过/拒绝 |
| 商品列表/详情 | GET /catalog/products, products/:id | 公开 |
| 购物车 | GET/POST/PATCH/DELETE /cart, /cart/items | 增删改查、改数量 |
| 下单/支付 | POST /order, POST /order/:id/pay, GET /order | 创建订单、支付、我的订单 |
| 推荐关系 | GET /referral | 我的邀请码、一级/二级人数、关系列表 |
| 佣金 | GET /commission | 我的佣金记录 |
| 个人资料 | GET /user/profile | 手机、邮箱、邀请码等 |
| **积分** | GET /user/points | 当前余额 + 积分明细 |
| **优惠券** | GET /user/coupons, POST /user/coupons/claim | 已领取列表、按券码领取 |
| **提现** | GET /user/withdrawals, POST /user/withdrawals | 可提现余额、申请记录、提交申请 |

数据库与配置已支持：两级推荐、佣金结算与冻结、提现规则（最低金额、每日上限、手续费）、VIP、优惠券、系统设置等；积分表 PointsLedger 已存在，写入可由后续业务（如订单完成送积分）接入。

## 三、管理后台（已做）

| 模块 | 功能 |
|------|------|
| 仪表盘 | 统计卡片（订单/用户/商品/GMV）+ 加载/刷新 |
| 用户管理 | 用户列表（手机、角色、审核、邀请码、注册时间） |
| 商品管理 | 列表、新增、编辑、上架/下架、分类 |
| 订单管理 | 订单列表、标记完成（触发佣金结算） |
| 佣金管理 | 佣金列表（用户、订单号、级别、金额、状态） |
| 提现管理 | 提现列表、通过/拒绝 |

## 四、用户端（已做）

| 页面/功能 | 说明 |
|-----------|------|
| 首页 | 品牌、精选商品卡片、平台说明、导航（首页/商品/购物车/登录/个人中心/管理后台） |
| 全部商品 | 商品列表、进详情 |
| 商品详情 | 名称、价格、描述、库存、加入购物车（需登录）、登录引导 |
| 登录/注册 | /login、/register，存 user_token，支持 redirect |
| 购物车 | /cart，列表、改数量、删除、合计；未登录跳 /login?redirect=/cart |
| **个人中心** /me | 个人资料（邀请码）、推荐（一级/二级人数）、佣金列表、积分（余额+明细）、优惠券（券码领取+已领取）、我的订单、提现（可提现余额+申请+记录） |

## 六、总结

- **推荐、积分、优惠券、提现、佣金**：后端接口与前台/后台对应页面已具备；积分目前仅展示与明细，发放规则可后续接订单/活动。
- **两级推荐、佣金仅订单完成结算、提现审核**：逻辑与后台操作已支持。
- 未做部分主要为：积分自动发放、下单选券与抵扣、VIP 购买与权益、内容页、系统设置与审计导出；可按需在现有结构上补齐。
